# Review Lenses

이 문서는 초안 이후 품질 향상을 위한 검토 렌즈 묶음이다.
기본값은 다 고쳐주기가 아니라, 무엇을 먼저 고쳐야 하는지 분명히 보여주는 것이다.

## 검토 순서
좋은 검토는 보통 아래 순서를 따른다.

1. 목적 정렬
2. 구조
3. 논리/근거
4. 독자 이해도
5. 문장
6. 문체

문장만 다듬는 검토는 가장 마지막에 온다.

## 4개 핵심 렌즈

### 1. Structural Editor
구조와 논리의 틀을 본다.

#### 보는 항목
- 핵심 메시지가 한 문장으로 보이는가?
- 도입이 제 역할을 하는가?
- 본론 순서가 가장 설득력 있는가?
- 결론이 흐려지지 않는가?

#### 자주 잡아내는 문제
- 도입이 길고 본론 진입이 늦다
- 논지가 겹치는 문단이 있다
- 결론에서 새 주장을 추가한다

### 2. Reader Lens
독자가 어디서 멈추는지 본다.

#### 보는 항목
- 어디서 지루해지는가?
- 어떤 문장이 어렵거나 추상적인가?
- 그래서 나와 무슨 상관에 답하는가?
- 끝까지 읽을 이유가 있는가?

#### 자주 잡아내는 문제
- 독자 효용이 초반에 보이지 않는다
- 설명은 많지만 맥락이 멀다
- 난이도가 독자 수준보다 높다

### 3. Skeptic / Fact Lens
사실과 근거를 본다.

#### 보는 항목
- 큰 주장에 근거가 있는가?
- 날짜/수치/인용이 맞는가?
- 과도한 일반화가 없는가?
- 반대 사례를 전혀 무시하지 않았는가?

#### 자주 잡아내는 문제
- 근거 없는 강한 표현
- 수치나 시점의 출처 부재
- 반론 가능성을 전혀 다루지 않음

### 4. Voice Keeper
문체와 톤의 일관성을 본다.

#### 보는 항목
- 작성자다운가?
- 장르에 맞는가?
- 과장, 상투어, 광고체가 없는가?
- 목적 전달을 해치지 않는가?

#### 자주 잡아내는 문제
- 과장된 카피 톤
- 문체가 문단마다 흔들림
- 스타일 적용이 과해서 정보 전달이 약해짐

## 피드백 출력 형식
기본 출력은 아래 4구역으로 고정한다.

### 유지할 점
지금 버전의 강점

### 먼저 고칠 점
가장 큰 문제 1~3개

### 선택 개선점
고치면 좋아지지만 필수는 아닌 것

### 확인 질문
다음 수정 전에 확인할 것

## 우선순위 표기
문제가 여러 개일 때는 우선순위를 붙인다.

- **P1**: 지금 바로 고쳐야 함
- **P2**: 고치면 확실히 좋아짐
- **P3**: 선택 개선

### 예시 표기
- **P1** 서론이 문제 제기를 하기 전에 배경 설명으로 너무 오래 머뭅니다.
- **P1** 본론 2와 3의 논지가 겹쳐 중복이 생깁니다.
- **P2** 결론에서 새 주장을 추가해 초점이 흐려집니다.
- **P3** 사례 한 개를 더 넣으면 독자 이해가 쉬워집니다.

## 좋은 피드백 문장 기준

### 피해야 할 말
- 좀 더 자연스럽게 써보세요.
- 약간 어색합니다.
- 더 설득력 있게 고쳐보세요.

### 권장하는 말
- 서론이 문제 제기를 하기 전에 배경 설명으로 너무 오래 머뭅니다.
- 본론 2와 3의 논지가 겹쳐 중복이 생깁니다.
- 결론에서 새 주장을 추가해 초점이 흐려집니다.
- 독자가 얻는 실익이 도입부에 보이지 않습니다.

## 난이도 조절 검토
독자 수준에 따라 아래를 본다.
- 개념 설명이 충분한가
- 문장이 너무 길지 않은가
- 용어 정의가 필요한가
- 사례가 독자에게 가까운가

### 기본 난이도 옵션
- 학생/초심자
- 일반 직장인
- 관심 있는 일반 독자
- 실무자
- 전문가

## 제목/도입/결론 특화 검토

### 제목
- 정확한가
- 흥미를 끄는가
- 장르와 맞는가

### 도입
- 주제와 독자 이유를 빨리 보여주는가
- 너무 늦게 본론으로 들어가지 않는가

### 결론
- 반복만 하지 않는가
- 독자에게 남길 핵심을 선명히 남기는가

## 역개요 모드
글이 길고 산만하면 역개요 모드로 먼저 구조를 드러낸다.

### 역개요를 써야 하는 상황
- 글이 길고 산만할 때
- 어디를 줄여야 할지 모를 때
- 문단 순서가 애매할 때
- 초안이 뭔가 많긴 한데 하나로 안 묶일 때

### 역개요 출력 형식
- 문단별 역할 한 줄 요약
- 중복되는 부분
- 순서를 바꾸면 좋아질 부분
- 빠진 연결부
- 추천 새 구조

## 문장 다듬기 전에 반드시 볼 것
문장 손질을 시작하기 전에 아래를 확인한다.
- 구조 문제가 남아 있는가?
- 같은 내용을 두 번 말하고 있는가?
- 근거 없이 세게 말한 부분이 있는가?
- 설명 순서가 잘못되지는 않았는가?

구조가 망가져 있으면 문장 polish는 효과가 작다.

## 최종 피드백 질문
검토를 끝내고 나면 필요할 때 아래를 묻는다.
- 구조부터 고칠까요, 문장부터 고칠까요?
- 독자 난이도를 더 낮출까요?
- 근거를 더 보강할까요?
- 문체를 더 작성자답게 맞출까요?
- 이 버전을 유지하고 제목만 손볼까요?
