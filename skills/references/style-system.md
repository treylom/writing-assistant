# Style System

## 목적
문체를 감으로 맞추는 일이 아니라, 분석 가능한 규칙으로 바꾸기 위한 참조 문서다.
좋은 스타일 적용은 누구의 문장을 복제하는 것이 아니라, 작성자의 톤과 리듬을 살리면서도 목적에 더 맞게 조정하는 일에 가깝다.

## 받을 수 있는 입력
- 작성자가 직접 쓴 글 2~5개
- 브랜드/조직 톤앤매너 문서
- 기존 뉴스레터/칼럼/블로그 샘플
- 메모, 스레드, 짧은 포스트 묶음
- 참고하고 싶은 타인 글
- 금지 표현/선호 표현 목록

## 입력 우선순위
1. 작성자가 직접 쓴 글
2. 작성자가 만든 문체 가이드 파일
3. 조직/브랜드 문서
4. 타인 레퍼런스
5. 형용사 수준의 취향 설명

## Style Card 10차원 정의
| 차원 | 설명 | 측정 기준 | 예시 |
|---|---|---|---|
| 1. 목소리 | 전체 인상이 차분한지, 친근한지, 단호한지 본다. | 서술 태도, 감정 노출 정도, 독자에게 거는 압력의 세기 | 차분한 설명형 / 친근한 대화형 / 단호한 주장형 |
| 2. 관점 거리 | 화자가 앞에 서는지, 한 걸음 물러나 설명하는지 본다. | 1인칭 빈도, 자기 경험 개입 정도, 설명자 시점 유지 여부 | 1인칭 중심 / 설명자형 |
| 3. 문장 길이 | 문장이 짧고 리듬감 있는지, 길고 사유형인지 본다. | 평균 문장 길이, 쉼표·접속절 사용 빈도, 한 문장당 아이디어 수 | 짧은 문장 위주 / 보통 / 긴 사유형 |
| 4. 문단 호흡 | 문단이 짧게 끊기는지, 길게 이어지는지 본다. | 문단당 문장 수, 시각적 여백, 문단 전환 빈도 | 짧은 단락 / 긴 단락 |
| 5. 어휘 수준 | 쉬운 단어 중심인지, 개념어가 많은지 본다. | 추상명사 밀도, 전문용어 사용 빈도, 설명 보조 여부 | 쉬운 단어 중심 / 보통 / 개념어 비중 높음 |
| 6. 전환 습관 | 문단과 문장을 어떻게 잇는지 본다. | 연결어 반복 패턴, 전환 위치, 문단 첫 문장 습관 | 그런데 / 다만 / 즉 / 결국 |
| 7. 강조 방식 | 무엇으로 핵심을 부각하는지 본다. | 질문, 대조, 반복, 짧은 결론 문장 사용 비율 | 질문형 강조 / 대조 / 반복 / 짧은 결론 |
| 8. 사례 사용법 | 주장을 어떤 사례로 지지하는지 본다. | 경험·비유·데이터·인용 사용 비율, 사례 길이 | 경험 중심 / 비유 중심 / 데이터 중심 / 인용 중심 |
| 9. 오프닝/엔딩 | 어떻게 시작하고 끝내는지 본다. | 첫 문단 진입 방식, 마지막 문단 마감 방식 | 장면으로 시작 / 명제로 시작 / 질문으로 마감 / 요약으로 마감 |
| 10. 말버릇/금지어 | 자주 쓰는 표현과 피해야 할 표현을 관리한다. | 반복 표현 빈도, 군더더기, 상투어, 금지 목록 준수 여부 | 자주: "결국" / 금지: 과장 표현, 광고체 |

## Style Card 정규 파일 형식 (Canonical Format)

아래 형식이 **유일한 표준**이다. GPTs에서 만든 Style Card를 Claude Code에서도 그대로 재사용할 수 있고, 그 반대도 동일하다.

```markdown
# Style Card

- profile_name:
- source_samples:
- default_intensity: medium
- updated_at:

## 1. voice
- keep:
- avoid:

## 2. perspective_distance
- keep:
- avoid:

## 3. sentence_length
- keep:
- avoid:

## 4. paragraph_rhythm
- keep:
- avoid:

## 5. vocabulary_level
- keep:
- avoid:

## 6. transition_habits
- keep:
- avoid:

## 7. emphasis_patterns
- keep:
- avoid:

## 8. evidence_usage
- keep:
- avoid:

## 9. opening_ending_patterns
- keep:
- avoid:

## 10. verbal_habits_banned_phrases
- keep:
- avoid:

## Change Log
- YYYY-MM-DD: keep/discard | dimension | reason | note
```

### 포맷 규칙
- 각 차원은 `keep`(유지할 특성)과 `avoid`(피할 특성)으로 이분화
- `Change Log`는 autoresearch 피드백 이력 (append-only)
- `default_intensity`: 약/medium/강 중 택 1
- GPTs에서 내보낸 파일도 이 형식이면 Claude Code에서 그대로 로드 가능

## 페르소나 깎기 루프
기본 워크플로우는 아래 순서를 따른다.

```text
샘플 수집 → 10차원 분석 → Style Card 초안 → 적용 → 피드백 → 업데이트
```

### 1. 샘플 수집
- 가능하면 작성자 본인 글을 2~5개 모은다.
- 장르가 섞여 있으면 공통 습관과 장르별 편차를 분리해 본다.
- Claude Code 환경에서는 필요 시 Read로 참조 자료를 읽고, Glob으로 관련 샘플 위치를 찾는다.

### 2. 10차원 분석
- 각 샘플을 10차원으로 분해해 반복 패턴을 찾는다.
- 일시적 컨디션인지, 지속적 습관인지 구분한다.
- 타인 레퍼런스는 복제 대상이 아니라 고수준 특징 추출 대상으로만 쓴다.

### 3. Style Card 초안
- 반복적으로 나타나는 패턴만 초안에 넣는다.
- 확신이 낮은 항목은 잠정안으로 표시한다.
- 샘플 수와 버전을 메타데이터에 기록한다.

### 4. 적용
- 현재 글의 목적, 독자, 장르 규칙을 먼저 확인한다.
- 장르 규칙과 충돌하는 문체 습관은 완화하거나 제외한다.
- 기본 적용 강도는 중간 강도다.

### 5. 피드백
- 적용 결과를 보고 유지할 점, 어색한 점, 과장된 점을 모은다.
- 피드백은 10차원 중 어디에 해당하는지 매핑한다.
- 단일 작업에서만 나온 요구인지, 반복되는 요구인지 구분한다.

### 6. 업데이트
- 지속적으로 반복되는 피드백만 Style Card에 반영한다.
- 변경 이력 표에 날짜, 차원, 변경 내용, 이유를 남긴다.
- 업데이트 후 다시 적용해 드리프트가 줄었는지 확인한다.

## autoresearch 피드백 판정 규칙

### 1. 피드백 분류: 일회성 vs 지속적
- **일회성**: 특정 원고, 특정 매체, 특정 캠페인에만 필요한 요구
- **지속적**: 장르가 달라도 반복해서 나오는 요구, 작성자 고유 습관과 직접 연결되는 요구

판정 질문:
1. 다른 글에도 그대로 적용되는가?
2. 같은 지적이 2회 이상 반복되었는가?
3. 목적이 아니라 스타일 습관 자체를 겨냥한 피드백인가?

세 질문 중 2개 이상이 yes면 지속적으로 본다.

### 2. 차원 매핑 규칙
| 피드백 키워드 | 연결할 차원 |
|---|---|
| 차분하게, 더 부드럽게, 더 단호하게 | 1. 목소리 |
| 나를 더 드러내기, 거리감 줄이기, 설명자 톤 | 2. 관점 거리 |
| 문장이 길다, 끊어 써라, 호흡이 짧다 | 3. 문장 길이 |
| 문단이 답답하다, 너무 잘게 끊긴다 | 4. 문단 호흡 |
| 너무 어렵다, 개념어가 많다, 쉽게 써라 | 5. 어휘 수준 |
| 연결이 튄다, 전환이 어색하다 | 6. 전환 습관 |
| 강조가 약하다, 반복이 많다, 질문이 과하다 | 7. 강조 방식 |
| 사례가 부족하다, 비유 대신 데이터, 인용이 많다 | 8. 사례 사용법 |
| 도입이 약하다, 결말이 흐리다 | 9. 오프닝/엔딩 |
| 말버릇이 거슬린다, 금지 표현 위반 | 10. 말버릇/금지어 |

### 3. keep / discard 판정
- **keep**: 지속적 피드백이고, 10차원 중 특정 차원으로 명확히 매핑되며, 다음 원고에도 재사용 가치가 있는 경우
- **discard**: 일회성 요구이거나, 특정 주제·매체·마감에만 맞는 요구인 경우

간단한 판정 규칙:
- 반복됨 + 차원 매핑 가능 + 재사용 가능 → keep
- 일회성 + 맥락 한정 + 재사용 낮음 → discard

### 4. 변경 이력 기록 형식
```markdown
| 날짜 | 차원 | 변경 내용 | 이유 |
| 2026-03-28 | 3. 문장 길이 | 긴 문장 선호 → 보통 길이 선호로 조정 | 최근 3개 원고에서 "호흡이 길다" 피드백 반복 |
```

## 문체 적용 강도
- **약**: 문체 느낌만 반영, 목적과 가독성 최우선
- **중**: 단어 선택, 문장 호흡, 전환 습관까지 반영
- **강**: 구조적 버릇, 리듬, 서술 거리까지 적극 반영

기본값은 **중**이다.

## 작성자 글 vs 타인 레퍼런스

### 작성자 자신의 문체
가능한 한 가깝게 반영해도 된다.

### 타인 레퍼런스
문장을 흉내 내지 말고 아래만 추출한다.
- 톤
- 밀도
- 설명 방식
- 구조
- 리듬
- 사례 처리

즉, 복제보다 고수준 특성 이전을 한다.

## 샘플이 빈약할 때
아래 경우는 과적합을 피한다.
- 샘플이 1개뿐
- 길이가 매우 짧음
- 장르가 서로 크게 다름
- 작성자 실제 글보다 타인 레퍼런스가 많음

이때는 확신도를 낮게 보고, 현재 문체 규칙은 잠정안이라고 명시한다.

## 스타일 드리프트 체크
적용 후 아래를 확인한다.
- 작성자 같아 보이는가?
- 그러나 전달력은 좋아졌는가?
- 지나치게 흉내 내는 느낌은 없는가?
- 장르와 독자에 맞는가?
- 불필요한 상투어나 장식이 늘지 않았는가?

## 깊은 문체 분석 요청이 들어오면
1. 분석 계획 제시
2. 필요한 샘플과 제약 확인
3. 샘플 정밀 분석
4. Style Card + 세부 규칙 + 금지 습관 정리
5. 재사용 가능한 텍스트/마크다운 가이드로 출력

## 추천 출력 형식

### 간단 버전
- 이 문체의 핵심 5가지
- 적용 시 주의점 3가지
- 지금 글에 반영한 방식

### 상세 버전
- Style Card
- 장르별 보정 규칙
- Before/After 예시
- 금지 표현 목록

## GPTs ↔ Claude Code 호환 보장
- Style Card 형식은 동일하게 유지한다.
- GPTs에서 내보낸 Style Card를 Claude Code에서 그대로 읽어 적용할 수 있다.
- Claude Code에서 업데이트한 Style Card도 같은 형식으로 다시 다른 도구에 전달할 수 있다.

## 기억할 점
문체는 마지막 광택이 아니라, 초안 단계부터 구조와 문장 리듬에 영향을 준다.
하지만 언제나 목적, 독자, 장르 규칙이 문체보다 위에 있다.
