# Prompt Templates

이 문서는 글쓰기 작업을 시작하거나 정교화할 때 바로 가져다 쓸 수 있는 프롬프트 템플릿 모음이다.
Claude Code 환경에 맞춰 참조 자료, Read, Write, Glob 같은 도구 사용 흐름과 함께 쓸 수 있게 정리했다.

## 사용 원칙
- 필요한 단계만 요청한다. 개요가 필요하면 초안까지 한 번에 요청하지 않는다.
- 참조 자료가 있으면 먼저 읽게 한다.
- 모르는 사실은 사실처럼 쓰지 않도록 명시한다.
- `/write` 커맨드와 함께 쓸 때는 목적, 독자, 장르, 분량, 제약을 먼저 넣는다.

## 1. 빠른 시작 템플릿
어떤 글을 쓰려고 하나요?
- 에세이 / 칼럼 / 기사 / 블로그 / 뉴스레터 / 보고서 / 기타

아래를 아는 범위에서만 적어주세요.
- 목적:
- 독자:
- 매체:
- 분량:
- 마감:
- 참조 자료/초안 유무:
- 문체 샘플 유무:
- 금지 표현:
- 지금 필요한 단계:

### /write 예시
```text
/write 아래 조건으로 글쓰기 작업을 시작해줘.
- 장르: 블로그
- 목적: 실무 팁 공유
- 독자: AI 도구를 처음 쓰는 직장인
- 분량: 1500자
- 참조 자료: notes.md
- 지금 필요한 단계: 개요
```

## 2. 막막할 때 방향안 요청
저는 [대략적 주제]로 [장르]를 써야 하는데 아직 방향이 막막합니다.

아래 형식으로 도와주세요.
1. 가능한 방향안 3개
2. 각 방향의 핵심 메시지 한 줄
3. 가장 추천하는 방향 1개
4. 그 방향으로 개요를 짜기 전에 필요한 질문 2개

## 3. 개요만 먼저 요청
아래 조건에 맞춰 **개요만** 작성해주세요. 본문은 아직 쓰지 말아주세요.

- 장르:
- 목적:
- 독자:
- 매체:
- 분량:
- 꼭 넣을 내용:
- 참조 자료:
- 원하는 톤:
- 금지 표현:

개요에는 아래를 포함해주세요.
- 제목 후보 3개
- 핵심 메시지 1줄
- 구조
- 파트별 예상 분량
- 필요한 근거/사례
- 아직 비어 있는 부분

### /write 예시
```text
/write 개요만 만들어줘. 본문은 쓰지 마.
- 장르: 뉴스레터
- 목적: 이번 호 구성안 설계
- 독자: 기존 구독자
- 참조 자료: issue-notes.md
```

## 4. 승인된 개요로 초안 쓰기
아래 개요를 바탕으로 초안을 작성해주세요.
개요를 크게 벗어나지 말고, 모르는 부분은 사실처럼 쓰지 말아주세요.

[승인된 개요 붙여넣기]

추가 조건:
- 톤:
- 독자 난이도:
- 분량:
- 살리고 싶은 표현:
- 피할 표현:

## 5. 구조만 고치기
아래 초안을 구조 중심으로만 고쳐주세요.
문장 polish는 크게 하지 말아주세요.

[초안 붙여넣기]

아래 형식으로 답해주세요.
- 유지할 점
- 먼저 고칠 점
- 구조 수정안
- 수정 후 개요

## 6. 논리와 근거 보강
아래 글에서 논리와 근거가 약한 부분만 찾아 보강해주세요.

[본문 붙여넣기]

형식:
- 약한 주장
- 왜 약한지
- 어떤 근거가 더 필요한지
- 보강 예시

## 7. 문장 다듬기 요청
아래 글을 뜻은 유지한 채 더 읽기 쉽게 다듬어주세요.
구조는 크게 바꾸지 말아주세요.

[본문 붙여넣기]

조건:
- 과장 금지
- 상투어 금지
- 문장만 다듬기
- 한국어 자연스러움 우선

## 8. 문체 카드 만들기
아래 샘플을 읽고, 재사용 가능한 `Style Card`를 만들어주세요.

[샘플 글 1]
[샘플 글 2]
[샘플 글 3]

형식:
- 한 줄 정체성
- 핵심 톤 3개
- 문장 호흡
- 자주 쓰는 장치
- 피해야 할 습관
- 오프닝/엔딩 특징
- 다른 장르에 적용할 때 주의점

### Claude Code 활용 팁
- 샘플이 파일이면 Read로 먼저 읽는다.
- 샘플 위치가 흩어져 있으면 Glob으로 모은 뒤 공통 패턴을 뽑는다.

## 9. 문체 적용하기
아래 `Style Card`를 먼저 읽고, 본문을 그 스타일에 맞게 다시 써주세요.

[Style Card]
...

[본문]
...

조건:
- 문체는 반영하되 목적 전달이 우선
- 장르 규칙은 유지
- 특정 문장을 복제하지 말 것

## 10. 참조 자료 기반 재구성
참조 자료를 먼저 읽고, 그 내용을 바탕으로 [장르] 형식으로 재구성해주세요.

조건:
- 목적:
- 독자:
- 매체:
- 분량:
- 강조 포인트:
- 제외 포인트:
- 원하는 톤:

순서는 아래로 진행해주세요.
1. 참조 자료 핵심 요약
2. 개요
3. 승인 질문

### /write 예시
```text
/write Read로 meeting-notes.md를 먼저 읽고, 그 내용을 바탕으로 보고서 개요를 만들어줘.
- 목적: 팀 공유
- 독자: 리더십
- 분량: 2쪽
```

## 11. 역개요 요청
아래 초안의 역개요를 뽑아주세요.

[초안 붙여넣기]

아래 형식으로 답해주세요.
- 문단별 역할 한 줄 요약
- 중복되는 부분
- 순서를 바꾸면 좋아질 부분
- 빠진 연결부
- 추천 새 구조

## 12. Quick Fact Check
아래 문장만 빠르게 검증해주세요.

[검증 대상 문장]

형식:
- 판정
- 근거 요약
- 핵심 출처
- 남는 불확실성

## 13. Structured Research
아래 주제로 구조화된 리서치를 해주세요.

- 주제:
- 목적:
- 핵심 질문:
- 독자:
- 최종 결과물 유형:

출력 형식:
- 요약
- 핵심 발견
- 상세 분석
- 반대 근거/예외
- 한계
- 참고 자료

## 14. 뉴스레터 구성안
이번 호 뉴스레터를 만들고 싶습니다.

- 주제:
- 독자:
- 발행 주기:
- 이번 호 목적:
- 포함 섹션:
- 참고자료:
- 문체 톤:

먼저 아래만 제시해주세요.
1. 이번 호 핵심 메시지
2. 오프닝 방향 2개
3. 전체 구성안
4. 아직 필요한 정보

## 15. 보고서 개요
아래 자료를 바탕으로 보고서 개요를 작성해주세요.

- 보고 목적:
- 보고 대상:
- 분량:
- 조직 양식 유무:
- 참고자료:
- 핵심 질문:
- 필요한 시각자료 유무:

개요에는 아래를 포함해주세요.
- Executive Summary 한 줄
- 구조
- 파트별 핵심 메시지
- 필요한 수치/근거
- 추가 확인이 필요한 부분
